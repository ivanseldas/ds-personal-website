Content Automation Workflow: From Telegram Prompt to Structured, Sourced Article

Turn a Telegram prompt into a structured, sourced article with an agentic RAG pipeline.

This project is an end-to-end automated system for generating long‑form, structured content based on user prompts. Built with n8n, it combines Telegram as a user interface, Supabase as a document vector database, and an Agentic RAG pipeline using LangChain with GPT‑4o‑mini.

Users send a topic or question via Telegram and receive a complete article draft in a consistent JSON schema. The workflow ensures outputs are grounded in real documents retrieved from Supabase and follow a technical writing style with citations and data awareness.

Workflow Overview

Workflow overview (diagram or screenshot)

The Problem

Producing consistent, long‑form content that is source‑grounded, technically styled, and structured for downstream use (apps, CMS, analytics) is hard to scale. This workflow automates research, drafting, and structuring while keeping outputs verifiable.

The Approach

The system orchestrates a multi‑step pipeline that handles session memory, document retrieval, prompt templating, and JSON‑structured generation.

  1. Telegram Bot — User Interface

    Users initiate generation by sending prompts to a Telegram bot. The workflow uses the message (or chat) ID to manage sessions and memory.

    Screenshot: Telegram input and result
  2. Supabase — Knowledge Store

    Documents live in PostgreSQL on Supabase and are indexed as vectors. On each request, a similarity search retrieves the most relevant chunks, injected into the prompt context.

    Screenshot: Supabase table with chunks + metadata
  3. n8n Workflow — Orchestration

    n8n manages memory, prompt templating, context merging, and post‑processing. LangChain BufferMemory keeps multi‑turn coherence. GPT‑4o‑mini receives structured context and outputs JSON.

Output Format Example

Responses returned to Telegram follow a consistent schema:

{
  "title": "How AI is Transforming Urban Planning",
  "summary": "This article explores how AI technologies, particularly generative and predictive models, are shaping the future of urban development.",
  "sections": [
    {
      "h2": "Introduction to Urban AI",
      "paras": [
        "AI is increasingly being integrated into the planning processes of cities worldwide...",
        "From traffic optimization to zoning analysis, its applications are broadening rapidly."
      ]
    },
    {
      "h2": "Key Technologies in Use",
      "paras": [
        "Machine learning models are being used to simulate population growth and resource needs...",
        "Geospatial data combined with predictive models enhances infrastructure planning."
      ]
    }
  ],
  "metrics": ["Projected cost savings", "Planning efficiency gains"],
  "citations": [
    { "section_path": "sections[0]", "quote": "AI is already helping cities model growth scenarios." }
  ],
  "missing_data": ["No data found for zoning impacts post-2023"]
}

Tech Stack

Workflow & Orchestration

  • n8n: flow control, retries, branching, post‑processing
  • LangChain: BufferMemory, RAG chain, prompt templates

Interfaces & Models

  • Telegram Bot API: prompt input and response delivery
  • OpenAI GPT‑4o‑mini: generation with structured JSON outputs

Knowledge & Retrieval

  • Supabase (PostgreSQL): document storage + metadata
  • Vector search: embeddings‑based similarity to ground responses

Key Results

  • Structured outputs: JSON schema ready for CMS or app integration
  • Grounded content: citations from Supabase‑retrieved documents
  • Agentic flow: multi‑turn memory and controlled prompt scaffolding
