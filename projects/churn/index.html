Telecom Churn Prediction: Flag At‑Risk Subscribers Early

Predict churn with high recall to power targeted, cost‑effective retention.

This project focuses on predicting customer churn in a telecom company using structured data and supervised learning models. It combines exploratory data analysis, feature engineering, and model evaluation to identify customers most likely to leave and support proactive retention strategies.

Project Overview

The dataset includes 5,000 customers with categorical and numerical features (e.g., international plan, voicemail usage, customer service calls, and call minutes). The goal is to classify churners with a focus on high recall.

Pipeline — Key Steps

  • Data Preprocessing: Handle categorical variables, remove non‑informative columns, and apply stratified sampling and class weighting to address imbalance.
  • Exploratory Data Analysis (EDA): Visualize churn patterns and feature relationships (distributions, correlation matrix).
  • Model Training & Evaluation: Compare Logistic Regression, Random Forest, and XGBoost. The best model, XGBoost, achieved a recall of 0.84, prioritizing correct identification of churners.

Business Impact

By ranking the top 500 customers at risk of churning, the model supports targeted retention campaigns. Based on validation metrics, this strategy could potentially reduce churn by 59.5% (95% CI) if acted upon effectively.

Tech Stack

  • Python, Pandas, scikit‑learn, XGBoost
  • Visualization: Matplotlib, Seaborn
  • Evaluation: Confusion matrix, ROC curve, classification report

Models were selected with a recall‑first objective to minimize false negatives (missed churners). Threshold tuning was explored to balance precision vs. recall for operational use.

Key Results

  • Recall 0.84 on validation with XGBoost; best overall for catching churners.
  • Actionable ranking of top‑risk customers for retention teams.
  • Clear diagnostics via feature importances and confusion matrices.
